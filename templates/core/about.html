How SkillForge is structured

Apps & architecture

core (home, about, dashboard), accounts (auth), courses (public CBVs), challenges (private CRUD), portfolio (main feature).

  • 10+ pages, 5+ CBVs, 5+ models, 5+ forms, responsive Bootstrap templates.
  • Public browse (courses) and private CRUD (challenges/submissions, portfolio).
  • Auth & roles: login/register; Staff group (limited), superuser (full).
  • Routing: namespaced URLs (courses:list, portfolio:detail), clean slugs/IDs.
Request lifecycle
URLconf → CBV → Form (validate) → ORM → Template (messages)
Highlights & conventions
  • Readable CBVs: slim views, most logic in forms/managers.
  • Template partials: shared base.html + toasts/messages partial.
  • Query hygiene: annotate counts/rating, avoid N+1s.
  • Ownership gates: users can only edit their own content; staff can moderate.
  • API-friendly: DRF patterns ready for integration/expansion.
Tech stack
  • Django 5
  • Python 3.13
  • Bootstrap 5
  • DRF (sample)
Data model
  • User / Course / Lesson
  • Challenge / Submission
  • Project / Review

Security & performance practices

Security
  • CSRF protection on forms and safe redirects.
  • Object-level permissions (review/project ownership; staff moderation).
  • Escaped templates and conservative exposure of user data.
  • Login required for private routes; read-only for public pages.
Performance & quality
  • Efficient ORM usage: select_related / prefetch_related.
  • Calculated annotations for counts/averages where useful.
  • Paginated lists and concise templates for fast rendering.
  • Clear URL naming and small, composable components.

python manage.py runserver
